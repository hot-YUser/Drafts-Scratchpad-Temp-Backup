# 開發信條

> **極致的快速、小巧的精緻、清晰的邏輯。**

以極客精神為傲,對「快速、小巧」有絕對且極致的渴望。東西越原生、越底層越好。

## 準則

- **越底層越好** — 能直呼 OS API(Win32 / ctypes / windows-sys)就不套框架;能用標準庫就不加依賴。零第三方依賴是理想狀態。
- **小與快是第一級指標** — 產物體積、啟動速度不是事後優化,是設計起點。願意為此承受更高維護成本(手寫 ctypes 結構、純 Win32 訊息迴圈、`opt-level="z"` + LTO + strip)。
- **邏輯清晰** — 短、直白、可讀。快速小巧 ≠ 潦草;不為壓行數而犧牲正確性。
- **誠實取捨** — 講清楚大小 / 啟動 / 依賴 / 是否原生,並標注為了小快所做的權衡。

## 實證:本專案的演進

Python + tkinter → ctypes 直呼 `GetAdaptersAddresses`(毫秒級偵測)→ Rust 純 Win32 重寫,**208 KB 單檔、零依賴、啟動即開**。
